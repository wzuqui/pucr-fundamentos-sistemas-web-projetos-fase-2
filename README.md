# Introdução

O Projeto da disciplina consiste na criação de uma plataforma online para um petshop, conforme abordado durante as aulas.
A plataforma online é um sistema web de comércio de produtos e serviços de um petshop.

Os produtos disponíveis estão divididos em 3 categorias: acessórios (roupas, brinquedos, camas, etc.), rações não perecíveis e produtos de higiene e limpeza (tapete higiênico, fraldas, etc.).

Você tem a liberdade de escolher as opções de apresentação, desde que haja pelo menos dois produtos de cada categoria.

Quanto aos serviços oferecidos pelo petshop são: banho e tosa com ou sem tele-busca.

Atenção: seu trabalho deve ser autoral e individual.

# Fase 1

## Enunciado da fase 1

Para a fase 1 do Projeto da Disciplina, você terá que desenvolver um sistema web para o petshop. Nesta fase, será realizada a construção das páginas sem o uso de customizações (CSS e Bootstrap). Também não deve haver a implementação de funções em JavaScript; isso será feito na fase 2.

Nesta etapa, elabore um documento contendo:
• Objetivo e metas: o que se quer alcançar e as atividades específicas para atingir o objetivo. Obs.: Para desenvolver esta parte, consulte as Aula 1 a 5.
• Link para os códigos-fonte (HTML) no repositório GitHub.
• Link para o sistema resultante (GitHub Pages ou similar).

A estrutura básica do seu sistema web, nesta etapa, deve conter:
• Cabeçalho (header): com informações do petshop, links de navegação para as páginas, etc. Obs.: Para desenvolver esta parte, consulte as Aula 1 a 5.
• Corpo das páginas (body): Visualização de dois produtos de cada uma das 3 categorias (foto, descrição, valor) e os serviços oferecidos pelo petshop (descrição, valor). Obs.: Para desenvolver esta parte, consulte as Aula 1 a 5.
• Rodapé (footer): contendo informações legais de autoria, contato, links de navegação, etc. Obs.: Para desenvolver esta parte, consulte as Aula 1 a 5.

ATENÇÃO: seu projeto deve ser estruturado com base nos requisitos de desenvolvimento do sistema para um petshop listados abaixo. Os requisitos mínimos do sistema web de front-end são: visualização das categorias (produtos) com imagens, descrição e valor, além dos serviços (descrição, valor e com tele-busca ou não) disponíveis no petshop.

O projeto deverá ser publicado na web, para isso deverá ser criada uma conta no GitHub e feita a postagem de todos os arquivos de código-fonte, o site e um arquivo de ajuda, descrevendo as funcionalidades da página web. Além disso, deve ser utilizado o GitHub Pages (ou similar) para hospedar o site do projeto do petshop diretamente por meio de um repositório do GitHub.com (para mais informações, acesse: https://docs.github.com/pt/pages/getting-started-with-github-pages/creating-a-github-pages-site).

O que você deve entregar: Um arquivo no formato PDF, conforme template fornecido pelos professores, contendo o link do GitHub utilizado, bem como do GitHub Pages.

Quais aulas você deve assistir para realizar a fase 1 do projeto: Aulas 1 a 5.

# Fase 2

## Enunciado da fase 2

Nesta fase, continue o desenvolvimento do seu sistema web para um petshop, conforme o que já foi elaborado na fase 1, observando os prazos do cronograma. O código deve ser comentado e estruturado em arquivos, conforme as aulas desta disciplina. Realize os ajustes necessários para minimizar problemas maiores antes da publicação. Descreva os ajustes que foram realizados.

Na fase 2, deve-se usar CSS/Bootstrap e implementar funções em JavaScript para que o sistema web do petshop se torne mais atrativo e dinâmico. Além disso, nesta fase, deve ser incluído o formulário de cadastro do cliente e do pet, assim como a forma de agendamento para tele-busca ou entrega do pet no local. Seguem os requisitos desta fase:

Utilização de CSS/Bootstrap (carrossel) e uso de JavaScript: o objetivo é tornar o sistema web mais atrativo e dinâmico (carrossel, funções temporais, etc.). Obs.: Para desenvolver esta parte, consulte a Aulas 7, 8 e 9.
Formulário de cadastro do cliente e do pet: elabore um formulário contendo os dados de identificação do cliente, tais como: nome, endereço, CPF, sexo, telefone, e-mail, etc. No formulário também deverá ter campos para identificação do pet, tais como: nome, raça e idade. Utilize elementos variados no formulário e atributos, como: input, check box, number, e-mail, radio button, placeholder, campos obrigatórios, etc.). Obs.: Para desenvolver esta parte, consulte a Aula 6. (aula 6 fala em enviar o formulario https://formsubmit.co)
Escolha do serviço e método de agendamento: o petshop tem 2 serviços, banho e tosa. O agendamento pode ser feito da seguinte forma: tele-busca ou entrega do pet no local (ambos com agendamento). Deve-se usar calendário para definir o dia e horário de agendamento. Obs.: Para desenvolver esta parte, consulte a Aula 6.
Acessibilidade: adaptar o sistema web do petshop desenvolvido na fase 1 para atender pelo menos um requisito de acessibilidade para deficientes visuais. Exemplo: uso do atributo “alt” para a audiodescrição de imagens. Obs.: Para desenvolver esta parte, consulte a Aula 10.

O projeto deverá ser publicado na web, para isso deverá ser criada uma conta no GitHub e feita a postagem de todos os arquivos de código-fonte, o site e um arquivo de ajuda, descrevendo as funcionalidades da página web. Além disso, deve ser utilizado o GitHub Pages (ou similar) para hospedar o site do projeto do petshop diretamente por meio de um repositório do GitHub.com (para mais informações, acesse: https://docs.github.com/pt/pages/getting-started-with-github-pages/creating-a-github-pages-site.

Quais aulas você deve assistir para realizar a fase 2 do projeto: Aulas 6 a 10.
